## 2. Descrição

O estudante inicia um percurso com a bicicleta reservada, ativando o registo de dados de localização, distância e tempo pelo Sistema GPS.

---

## 3. Pré-condições

- O estudante está autenticado na plataforma (UC-02).
- O estudante tem uma reserva ativa.
- A bicicleta está disponível no local indicado.
- O Sistema GPS está operacional.

---

## 4. Pós-condições

- O percurso fica com estado **ativo**.
- O Sistema GPS inicia o registo de dados em tempo real.
- A bicicleta passa ao estado **em utilização**.
- A reserva associada fica encerrada.

---

## 5. Fluxo Principal

| Passo | Ator | Ação |
|-------|------|------|
| 1 | Estudante | Acede à sua reserva ativa na plataforma. |
| 2 | Sistema | Apresenta os detalhes da reserva e a opção de iniciar percurso. |
| 3 | Estudante | Seleciona a opção de iniciar percurso. |
| 4 | Sistema | Verifica que a reserva está ativa e dentro do prazo. |
| 5 | Sistema | Regista o timestamp de início do percurso. |
| 6 | Sistema | Altera o estado da bicicleta para *em utilização*. |
| 7 | Sistema | Invoca o Sistema GPS para iniciar o registo de dados. |
| 8 | Sistema GPS | Confirma início do registo e começa a enviar dados de localização. |
| 9 | Sistema | Apresenta ao estudante o percurso ativo com dados em tempo real. |

---

## 6. Fluxos Alternativos

### FA-01 — Reserva expirada (passo 4)
1. O sistema deteta que a reserva expirou (mais de 30 minutos).
2. O sistema apresenta a mensagem: *"A tua reserva expirou. Por favor, faz uma nova reserva."*
3. O estado da bicicleta volta a *disponível*.
4. O caso de uso termina sem iniciar percurso.

### FA-02 — Bicicleta não se encontra no local (passo 4)
1. O estudante indica que a bicicleta não está no local indicado.
2. O sistema regista a ocorrência e notifica o administrador.
3. O caso de uso termina sem iniciar percurso.

---

## 7. Exceções

### EX-01 — Sistema GPS indisponível (passo 7)
1. O sistema não consegue contactar o Sistema GPS.
2. O sistema regista o erro internamente.
3. O sistema apresenta a mensagem: *"Não foi possível ativar o registo GPS. O percurso será registado sem dados de localização."*
4. O percurso inicia sem registo GPS.

---

## 8. User Story

> Como **estudante do IPBeja**, quero **iniciar um percurso com a bicicleta reservada**, para **registar a minha deslocação e monitorizar os meus indicadores de saúde e eficiência energética**.

---

## 9. Critérios de Aceitação

**Cenário 1 — Início bem-sucedido**

**Dado que** o estudante tem reserva ativa e a bicicleta está disponível,  
**Quando** seleciona iniciar percurso,  
**Então** o sistema deve registar o início, ativar o GPS e apresentar o percurso ativo.

---

**Cenário 2 — Reserva expirada**

**Dado que** a reserva do estudante expirou,  
**Quando** tenta iniciar o percurso,  
**Então** o sistema deve informar que a reserva expirou e não iniciar o percurso.

---

**Cenário 3 — GPS indisponível**

**Dado que** o Sistema GPS não está operacional,  
**Quando** o estudante inicia o percurso,  
**Então** o sistema deve iniciar o percurso sem GPS e informar o estudante.
