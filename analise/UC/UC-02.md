## 2. Descrição

O utilizador (Estudante ou Administrador) autentica-se na plataforma U-Bike com as suas credenciais institucionais, obtendo acesso às funcionalidades correspondentes ao seu perfil.

---

## 3. Pré-condições

- O utilizador possui uma conta registada e ativa na plataforma.
- O utilizador possui credenciais institucionais válidas (@ipbeja.pt).
- O Sistema de Autenticação Institucional está disponível.

---

## 4. Pós-condições

- O utilizador fica com sessão iniciada na plataforma.
- O sistema redireciona o utilizador para a página inicial correspondente ao seu perfil (Estudante ou Administrador).
- É registado o timestamp do último acesso.

---

## 5. Fluxo Principal

| Passo | Ator | Ação |
|-------|------|------|
| 1 | Utilizador | Acede à página de login da plataforma U-Bike. |
| 2 | Utilizador | Introduz o email institucional e a password. |
| 3 | Utilizador | Submete o formulário de login. |
| 4 | Sistema | Valida o formato dos dados introduzidos. |
| 5 | Sistema | Envia as credenciais para o Sistema de Autenticação Institucional. |
| 6 | Sistema de Autenticação Institucional | Valida as credenciais e devolve confirmação. |
| 7 | Sistema | Identifica o perfil do utilizador (Estudante ou Administrador). |
| 8 | Sistema | Inicia a sessão e redireciona para a página inicial do respetivo perfil. |

---

## 6. Fluxos Alternativos

### FA-01 — Credenciais incorretas (passo 6)
1. O Sistema de Autenticação Institucional rejeita as credenciais.
2. O sistema incrementa o contador de tentativas falhadas.
3. O sistema apresenta a mensagem: *"Email ou password incorretos. Por favor, tente novamente."*
4. O utilizador pode tentar novamente (retorna ao passo 2).

### FA-02 — Conta bloqueada após tentativas falhadas (passo 6)
1. O utilizador atinge o limite de 5 tentativas falhadas consecutivas.
2. O sistema bloqueia temporariamente a conta por 15 minutos.
3. O sistema apresenta a mensagem: *"Conta temporariamente bloqueada por excesso de tentativas. Tente novamente mais tarde."*
4. O caso de uso termina.

### FA-03 — Conta não confirmada (passo 7)
1. O sistema deteta que a conta existe mas ainda não foi confirmada por email.
2. O sistema apresenta a mensagem: *"A sua conta ainda não foi confirmada. Verifique o seu email."*
3. O sistema oferece a opção de reenviar o email de confirmação.
4. O caso de uso termina sem iniciar sessão.

### FA-04 — Utilizador opta por recuperar password (passo 2)
1. O utilizador clica em *"Esqueci-me da password"*.
2. O sistema estende para UC-03 — Recuperar password.

---

## 7. Exceções

### EX-01 — Sistema de Autenticação Institucional indisponível (passo 5)
1. O sistema não consegue contactar o Sistema de Autenticação Institucional.
2. O sistema apresenta a mensagem: *"Serviço de autenticação temporariamente indisponível. Por favor, tente novamente mais tarde."*
3. O caso de uso termina sem iniciar sessão.

### EX-02 — Timeout na resposta (passo 6)
1. O Sistema de Autenticação Institucional não responde dentro do tempo limite (5 segundos).
2. O sistema regista o erro internamente.
3. O sistema apresenta a mensagem de indisponibilidade e termina o caso de uso.

---

## 8. User Story

> Como **utilizador registado na plataforma U-Bike**, quero **autenticar-me com as minhas credenciais institucionais**, para **aceder às funcionalidades da plataforma de forma segura**.

---

## 9. Critérios de Aceitação

**Cenário 1 — Autenticação bem-sucedida**

**Dado que** o utilizador possui conta ativa e credenciais corretas,  
**Quando** submete o formulário de login,  
**Então** o sistema deve iniciar a sessão e redirecionar para a página inicial do seu perfil.

---

**Cenário 2 — Credenciais incorretas**

**Dado que** o utilizador introduz uma password errada,  
**Quando** submete o formulário de login,  
**Então** o sistema deve apresentar mensagem de erro e não iniciar sessão.

---

**Cenário 3 — Bloqueio por tentativas excessivas**

**Dado que** o utilizador falhou a autenticação 5 vezes consecutivas,  
**Quando** tenta autenticar-se novamente,  
**Então** o sistema deve bloquear o acesso temporariamente por 15 minutos e informar o utilizador.

---

**Cenário 4 — Redirecionamento por perfil**

**Dado que** existem dois perfis (Estudante e Administrador),  
**Quando** cada um autentica com sucesso,  
**Então** o sistema deve redirecionar cada um para a respetiva página inicial com as funcionalidades do seu perfil.
