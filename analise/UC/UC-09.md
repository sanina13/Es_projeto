## 2. Descrição

O estudante consulta o histórico dos seus percursos realizados, podendo visualizar detalhes individuais de cada viagem, incluindo dados de distância, duração, percurso no mapa, calorias estimadas e CO₂ poupado.

---

## 3. Pré-condições

- O estudante está autenticado na plataforma (UC-02).
- O estudante tem pelo menos um percurso com estado **concluído**.

---

## 4. Pós-condições

- O sistema apresenta a lista de percursos realizados pelo estudante.
- Nenhum dado é alterado (operação de leitura).

---

## 5. Fluxo Principal

| Passo | Ator | Ação |
|-------|------|------|
| 1 | Estudante | Acede à secção "Histórico de Percursos" na plataforma. |
| 2 | Sistema | Obtém a lista de percursos concluídos do estudante, ordenados por data (mais recente primeiro). |
| 3 | Sistema | Apresenta a lista com informação resumida de cada percurso (data, origem, destino, distância, duração). |
| 4 | Estudante | Seleciona um percurso da lista para ver detalhes. |
| 5 | Sistema | Apresenta os detalhes completos do percurso selecionado: mapa do trajeto, distância total, duração, velocidade média, calorias estimadas, CO₂ poupado e tipo de bicicleta utilizada. |

---

## 6. Fluxos Alternativos

### FA-01 — Estudante sem percursos registados (passo 2)
1. O sistema deteta que o estudante não tem percursos concluídos.
2. O sistema apresenta a mensagem: *"Ainda não realizaste nenhum percurso. Reserva uma bicicleta para começar!"*
3. O sistema sugere um atalho para o UC-04 (Reservar bicicleta).
4. O caso de uso termina.

### FA-02 — Filtrar percursos (passo 3)
1. O estudante aplica filtros à lista de percursos (por intervalo de datas, por bicicleta, por ponto de partida/chegada).
2. O sistema atualiza a lista de acordo com os filtros aplicados.
3. O fluxo continua no passo 4.

### FA-03 — Exportar histórico (passo 3)
1. O estudante seleciona a opção de exportar o histórico.
2. O sistema gera um ficheiro (PDF ou CSV) com a lista de percursos e respetivos indicadores.
3. O sistema disponibiliza o ficheiro para download.

---

## 7. Exceções

### EX-01 — Dados de percurso incompletos (passo 5)
1. O sistema deteta que o percurso selecionado tem dados incompletos (ex: sem dados GPS).
2. O sistema apresenta os dados disponíveis e indica: *"Alguns dados deste percurso estão indisponíveis devido a falha no registo GPS."*
3. O mapa do trajeto não é apresentado, mas os restantes dados (duração, distância estimada) são mostrados.

---

## 8. User Story

> Como **estudante do IPBeja**, quero **consultar o histórico dos meus percursos de bicicleta**, para **acompanhar as minhas deslocações e rever os dados de cada viagem**.

---

## 9. Critérios de Aceitação

**Cenário 1 — Consulta bem-sucedida**

**Dado que** o estudante tem percursos concluídos,  
**Quando** acede ao histórico de percursos,  
**Então** o sistema deve apresentar a lista ordenada por data com informação resumida de cada percurso.

---

**Cenário 2 — Detalhes de um percurso**

**Dado que** o estudante está na lista de percursos,  
**Quando** seleciona um percurso específico,  
**Então** o sistema deve apresentar os detalhes completos incluindo mapa, distância, duração, calorias e CO₂ poupado.

---

**Cenário 3 — Sem percursos registados**

**Dado que** o estudante não tem percursos concluídos,  
**Quando** acede ao histórico,  
**Então** o sistema deve informar que não existem percursos e sugerir a reserva de uma bicicleta.

---

**Cenário 4 — Filtrar percursos**

**Dado que** o estudante tem múltiplos percursos concluídos,  
**Quando** aplica filtros por intervalo de datas,  
**Então** o sistema deve apresentar apenas os percursos dentro do intervalo selecionado.
