## 2. Descrição

O estudante termina o percurso em curso, finalizando o registo de dados de localização, distância, tempo e calorias estimadas. O sistema calcula automaticamente os indicadores de eficiência energética e de saúde associados ao percurso.

---

## 3. Pré-condições

- O estudante está autenticado na plataforma (UC-02).
- O estudante tem um percurso com estado **ativo** (UC-06).
- A bicicleta está no estado **em utilização**.

---

## 4. Pós-condições

- O percurso fica com estado **concluído**.
- Os dados do percurso são persistidos (distância, duração, velocidade média, calorias, CO₂ poupado).
- O Sistema GPS termina o registo de dados.
- A bicicleta passa ao estado **aguarda devolução**.
- Os indicadores pessoais do estudante são atualizados (UC-10).

---

## 5. Fluxo Principal

| Passo | Ator | Ação |
|-------|------|------|
| 1 | Estudante | Acede ao percurso ativo na plataforma. |
| 2 | Sistema | Apresenta os dados em tempo real do percurso (distância, duração, localização). |
| 3 | Estudante | Seleciona a opção de terminar percurso. |
| 4 | Sistema | Solicita confirmação ao estudante. |
| 5 | Estudante | Confirma a terminação do percurso. |
| 6 | Sistema | Regista o timestamp de fim do percurso. |
| 7 | Sistema | Invoca o Sistema GPS para parar o registo de dados. |
| 8 | Sistema GPS | Confirma a paragem e envia os dados finais do percurso. |
| 9 | Sistema | Calcula os indicadores do percurso (distância total, duração, velocidade média, calorias estimadas, CO₂ poupado). |
| 10 | Sistema | Altera o estado do percurso para *concluído*. |
| 11 | Sistema | Altera o estado da bicicleta para *aguarda devolução*. |
| 12 | Sistema | Atualiza os indicadores pessoais acumulados do estudante. |
| 13 | Sistema | Apresenta ao estudante o resumo do percurso com os indicadores calculados. |

---

## 6. Fluxos Alternativos

### FA-01 — Estudante cancela a confirmação (passo 5)
1. O estudante cancela a terminação do percurso.
2. O sistema mantém o percurso no estado **ativo**.
3. O caso de uso retorna ao passo 2.

### FA-02 — Percurso sem dados GPS (passo 8)
1. O percurso foi iniciado sem GPS (ver EX-01 do UC-06).
2. O sistema calcula os indicadores apenas com base na duração e na distância estimada entre pontos de levantamento e devolução.
3. O fluxo continua no passo 9 com dados parciais.

---

## 7. Exceções

### EX-01 — Falha de comunicação com o Sistema GPS (passo 7)
1. O sistema não consegue contactar o Sistema GPS para finalizar o registo.
2. O sistema regista o erro internamente.
3. O sistema utiliza os últimos dados recebidos do GPS para calcular os indicadores.
4. O percurso é concluído com os dados disponíveis.
5. O sistema apresenta a mensagem: *"O percurso foi terminado, mas alguns dados de localização podem estar incompletos."*

### EX-02 — Tempo máximo de percurso excedido (passo 1)
1. O sistema deteta que o percurso ativo ultrapassou o tempo máximo permitido (ex: 4 horas).
2. O sistema termina automaticamente o percurso.
3. O sistema notifica o estudante e o administrador.
4. O percurso é registado com estado **concluído (tempo excedido)**.

---

## 8. User Story

> Como **estudante do IPBeja**, quero **terminar o meu percurso de bicicleta**, para **visualizar o resumo da viagem e acompanhar os meus indicadores de saúde e eficiência energética**.

---

## 9. Critérios de Aceitação

**Cenário 1 — Terminação bem-sucedida**

**Dado que** o estudante tem um percurso ativo com GPS operacional,  
**Quando** seleciona terminar percurso e confirma,  
**Então** o sistema deve registar os dados finais, calcular os indicadores e apresentar o resumo do percurso.

---

**Cenário 2 — Terminação sem dados GPS**

**Dado que** o percurso foi iniciado sem registo GPS,  
**Quando** o estudante termina o percurso,  
**Então** o sistema deve calcular indicadores com dados parciais e informar o estudante.

---

**Cenário 3 — Tempo máximo excedido**

**Dado que** o percurso ativo ultrapassou o tempo máximo permitido,  
**Quando** o sistema deteta a situação,  
**Então** o percurso deve ser terminado automaticamente e o estudante e administrador notificados.
