UC-03 — Recuperar Password
Projeto: U-Bike IPBeja
UC: Engenharia de Software 2025/2026
Fase: Análise — Especificação de Caso de Uso

1. Identificação
CampoDetalheIDUC-03NomeRecuperar passwordAtor principalEstudante / AdministradorAtores secundáriosSistema de Autenticação InstitucionalVersão1.0Data2025/2026

2. Descrição
O utilizador que não se recorda da sua password solicita a recuperação da mesma através do email institucional, recebendo um link temporário que lhe permite definir uma nova password.

3. Pré-condições

O utilizador possui uma conta registada e ativa na plataforma.
O utilizador possui acesso ao email institucional associado à conta.
O Sistema de Autenticação Institucional está disponível.


4. Pós-condições

A password do utilizador é redefinida com sucesso.
O link de recuperação utilizado fica inválido.
O utilizador consegue autenticar-se com a nova password.


5. Fluxo Principal
PassoAtorAção1UtilizadorNa página de login, clica em "Esqueci-me da password".2SistemaApresenta o formulário de recuperação de password.3UtilizadorIntroduz o email institucional associado à conta.4SistemaValida o formato do email introduzido.5SistemaVerifica se o email existe na base de dados.6SistemaGera um link de recuperação único e temporário (válido por 1 hora).7SistemaEnvia o link de recuperação para o email institucional do utilizador.8SistemaApresenta a mensagem: "Email de recuperação enviado. Verifique a sua caixa de entrada."9UtilizadorAcede ao email e clica no link de recuperação.10SistemaValida o link (existência e prazo de validade).11SistemaApresenta o formulário para definir nova password.12UtilizadorIntroduz a nova password e confirma.13SistemaValida os requisitos da nova password.14SistemaAtualiza a password na base de dados e invalida o link de recuperação.15SistemaApresenta mensagem de sucesso e redireciona para a página de login.

6. Fluxos Alternativos
FA-01 — Email não encontrado na base de dados (passo 5)

O sistema não encontra o email introduzido.
Por razões de segurança, o sistema apresenta a mesma mensagem de sucesso do passo 8, sem revelar se o email existe ou não.
O caso de uso termina sem enviar email.

FA-02 — Nova password não cumpre requisitos (passo 13)

O sistema deteta que a nova password não cumpre os requisitos mínimos.
O sistema apresenta os requisitos em falta.
O utilizador corrige a password (retorna ao passo 12).

FA-03 — Nova password igual à anterior (passo 13)

O sistema deteta que a nova password é igual à password atual.
O sistema apresenta a mensagem: "A nova password não pode ser igual à anterior."
O utilizador introduz uma password diferente (retorna ao passo 12).


7. Exceções
EX-01 — Link de recuperação expirado (passo 10)

O utilizador clica num link expirado (mais de 1 hora).
O sistema apresenta a mensagem: "O link de recuperação expirou. Por favor, solicite um novo."
O sistema redireciona para o formulário de recuperação (retorna ao passo 2).

EX-02 — Link de recuperação já utilizado (passo 10)

O utilizador tenta utilizar um link já usado anteriormente.
O sistema apresenta a mensagem: "Este link já foi utilizado e não é válido."
O sistema redireciona para o formulário de recuperação (retorna ao passo 2).

EX-03 — Falha no envio do email (passo 7)

O sistema não consegue enviar o email de recuperação.
O sistema regista o erro internamente.
O sistema apresenta a mensagem: "Não foi possível enviar o email. Por favor, tente novamente mais tarde."
O caso de uso termina.


8. User Story

Como utilizador registado que esqueceu a password, quero recuperar o acesso à minha conta através do email institucional, para voltar a utilizar a plataforma U-Bike sem perder os meus dados.


9. Critérios de Aceitação
Cenário 1 — Recuperação bem-sucedida
Dado que o utilizador tem conta ativa e acesso ao email institucional,
Quando solicita recuperação de password e segue o link recebido,
Então o sistema deve permitir definir uma nova password e redirecionar para o login.

Cenário 2 — Email não registado
Dado que o utilizador introduz um email que não existe na plataforma,
Quando submete o formulário de recuperação,
Então o sistema deve apresentar a mensagem de sucesso sem revelar se o email existe, por razões de segurança.

Cenário 3 — Link expirado
Dado que o utilizador clica no link de recuperação após 1 hora,
Quando tenta aceder ao formulário de nova password,
Então o sistema deve informar que o link expirou e oferecer a opção de solicitar um novo.

Cenário 4 — Password igual à anterior
Dado que o utilizador tenta definir uma password igual à atual,
Quando submete o formulário de nova password,
Então o sistema deve rejeitar e solicitar uma password diferente.
